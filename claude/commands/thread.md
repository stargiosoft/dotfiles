## 출력 형식

### 본글 (1/3)

**총 5줄. 줄 바꿈 위치가 곧 리듬이야. 아래 구조를 그대로 따라.**

```
[후킹 1줄] — 반전이나 반박 형태. 30자 이내. 문장 하나로 끝.

[설명 1줄] — 왜 그런지 핵심 이유 하나.
[설명 1줄] — 구체적 사례나 수치.
[설명 1줄] — 반전 또는 대비 구도.

[궁금증 유발 1줄] — "그럼 뭐야?" 형태로 끊고, 댓글로 유도.
```

**후킹 1줄 예시 (이 느낌으로):**
- `"코딩 배워야 하나요?" 이 질문 자체가 틀렸어.`
- `리텐션 올리고 싶으면 기능 추가부터 하지 마.`
- `온보딩 10단계 갈아엎기 전에 이것만 알아.`

짧고, 반박하거나 뒤집는 구조. 설명 없이 딱 끊어.

---

### 댓글 구성 (2~5개)

댓글 수는 내용의 깊이에 따라 자유롭게 결정해. 단, 아래 규칙을 지켜.

**댓글 수 기준:**
- 2개: 핵심 공개 + 반론 선답으로 충분할 때
- 3개: 핵심 공개 + 실전 팁 + 반론 선답
- 4~5개: 사례, 단계별 설명, 추가 인사이트가 필요할 때

**각 댓글 공통 규칙:**
- 총 6줄 이내
- 한 줄에 문장 하나
- 마지막은 한 줄 마무리로 끊어

**자주 쓰는 댓글 유형 (골라서 조합):**

핵심 공개 — 본글에서 감춘 정보. 번호 리스트 3개 + 마무리 1줄.
```
1. 항목명
짧은 설명 1줄.

2. 항목명
짧은 설명 1줄.

3. 항목명
짧은 설명 1줄.

[마무리 한 줄]
```

실전 팁 — 바로 쓸 수 있는 팁. 번호 리스트 또는 단계.
```
[상황 설정 1줄]
1순위 — ~
2순위 — ~
3순위 — ~
[마무리 한 줄]
```

반론 선답 — 독자가 가질 반박을 먼저 꺼내고 답해.
```
"[예상 반론]" ↓

[짧은 답 1줄]
[부연 또는 수치 1줄]
[마무리 한 줄]
```

추가 인사이트 — 본글과 다른 각도의 정보.
```
[새로운 시각 1줄]
[설명 2줄 이내]
[마무리 한 줄]
```

## 후킹 전략

| 전략 | 완성 문장 예시 |
|------|--------------|
| 반전/반박 | `"A 해야 하나요?" 이 질문 자체가 틀렸어.` |
| 손실 회피 | `이거 모르고 기획하면 리텐션 못 올려.` |
| 구체적 숫자 | `온보딩 3단계에서 유저 40%가 떠나는 이유 있어.` |
| 타겟 지목 | `기획자인데 이 개념 모르면 진짜 손해야.` |
| 문제점 자극 | `왜 전환율이 안 오르는지 이제 알 것 같아.` |
| 호기심 유발 | `유저가 앱 지우는 진짜 이유, 아무도 안 말해줘.` |

## 품질 체크 (출력 직전 확인)

- [ ] 첫 줄이 30자 이내이고, 반전/반박 구조인가
- [ ] 본글이 정확히 5줄인가 (빈 줄 제외)
- [ ] 댓글 1이 번호 리스트 3개 + 마무리 1줄인가
- [ ] 댓글 2가 6줄 이내인가
- [ ] 한 줄에 문장이 2개 이상 들어가지 않았는가
- [ ] 개념 이름을 나열하지 않고 이야기로 녹였는가
- [ ] '~했음' 어조, 이모티콘 없는가
